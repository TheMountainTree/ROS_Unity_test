### Task State
- A: IDLE: 待机、等待任务指令
- B: OBSERVE: 观察，获取环境感知数据
- C: SELECT: 从检测结果中选择目标对象
- D: PLAN: 生成可执行的运动轨迹计划
- E: EXECUTE: 执行规划好的任务

### eegbot_msgs
定义了项目所用到的节点通信的消息类型

| Feature            | msg(Message)          | srv(Service)                |
| :----------------- | :-------------------- | :-------------------------- |
| Communication Mode | Publisher/Subscriber  | Request/Response            |
| Direction          | one way               | two way                     |
| Connection Type    | one to many           | one to one                  |
| Usage Scenario     | Continous data stream | Occasional queries/commands |

### sim_core
 sim_core 是项目的仿真核心模块，负责机器人仿真环境的接口封装和控制。

1. PyBullet IK 接口 (pybullet_ik_interface22.py)
- 使用 PyBullet 进行逆运动学 (IK) 求解
- 将 6D 末端位姿转换为关节角度
- 支持相对于机械臂基座的位姿计算
2. ROS2 接口 (zmq_interface_with_ik27.py)
- 将 ManiSkill 仿真环境与 ROS2 通信桥接
- 发布话题
	- /fetch_head/rgb/image_raw - RGB 图像
    - /fetch_head/depth/image_raw - 深度图像
    - /robot_state - 机器人关节状态
- 提供服务
	- /set_joint_positions - 设置关节位置
3. 主循环 (main_loop.py)
- 创建 ManiSkill 仿真环境 (Fetch 机器人)
- 使用 pd_joint_pos 控制模式
- 持续运行仿真，等待外部控制指令
只提供仿真场景的基本组件

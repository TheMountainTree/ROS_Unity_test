ros2挂起ros tcp point,unity使用modules ros tcp connector,从而让ros和unity通过udp的形式通信，随后unity端可以监听ros端的节点

P300_Stimulus.cs和SSVEP_Stimulus.cs是两个trigger,分别对应unity组件中的stimulus_trigger和SSVEP_trigger，这两个trigger会自动寻找/image_seg节点和通信器

因此需要首先把ROS2UnityComponent组件（通信器）挂载到某个GameObject下或者挂载到某个trigger下，这样算是构建好了通信，随后只要ros端在/image_seg下发送六张图片，trigger组件会自动寻找node并订阅

在此基础上，
- ImageUIs确定了每个图片的内容，默认情况应该为RawImageX
- FrameUIs确定了闪烁的UI，cellX为图片单元闪烁，FilckerFrameX为图片为指示器闪烁
- Stimulus Panel定义了trigger对象，默认情况应为StimulusPanel

可以通过以下指令调整节点`/task_state`参数
```
ros2 topic pub /task_state std_msgs/msg/String "data: 'E'" --once
```

`dual_view_pipeline_service_with_pcd_save4.py`确认为直接python运行，而不是节点运行，因此所有图片会被保存到`eeg_robot/src/root_ctr/graph/graph/results`中去，而`/image_seg`话题由`pics_retrieval.py`文件发布。因此需要读取上述分割后的文件，使用`resolve graph results dir`和`get latest segmentation dir`两个函数自动获取（无需手动配置）最新分割后的文件。此外，注意ros/opencv的图像坐标系与unity不同，因此需要翻转。此外，listdir获取的数据是无序的，需要添加sorted排序。
`pics_retrieval.py`这个文件的启动需要`task state`处于`E`的阶段，unity获取到六张图片后会自动运行。

## 核心文件变更详细说明

### 1. CentralControllerSSVEPNode3.py - Pretrain Epoch 边界判定机制重构

#### 1.1 参数与协议约束调整

| 参数 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| `trigger_local_ip` | `"0.0.0.0"` | `"192.168.56.103"` | 绑定到 Ubuntu 实际网卡 IP |
| `eeg_fs` | `250.0` | `1000.0` | EEG 采样率调整为 1kHz |
| `eeg_frame_floats` 验证 | `>= eeg_n_channels` | `== eeg_n_channels + 1` | 必须包含 trigger 通道 |

#### 1.2 Epoch 边界判定机制变更

**旧方案**：基于时间窗口截断
- 在 `_enter_pretrain_stim()` 时记录 `stim_start_abs`
- 在 `_enter_pretrain_rest()` 时记录 `stim_end_abs`
- 使用 `_normalize_epoch_samples()` 截断或填充到固定长度

**新方案**：基于 TCP 数据流中的 trigger 通道值
- 新增 `_process_eeg_trigger_sample()` 方法，从 TCP 流逐采样解析 trigger
- trigger 语义：
  - `1`：epoch 开始边界
  - `2`：epoch 结束边界，按 inclusive 处理
- epoch 保存区间为 `[start_abs, end_abs + 1)`
- 移除 `target_samples` 和 `_normalize_epoch_samples()`
- trial 结束时保存真实长度的变长 epoch

#### 1.3 运行时字段与记录结构

新增字段：
```python
self.last_trigger_value = 0
self.epoch_start_pending = False
self.current_stim_enter_abs = -1
self.current_stim_exit_abs = -1
self.current_epoch_complete = False
self.current_trial_record_written = False
```

CSV 字段更新：
- `ssvep3_pretrain_trials_*.csv`
  - `stim_start_abs` -> `stim_enter_abs`
  - `stim_end_abs` -> `stim_exit_abs`
  - 新增 `epoch_start_abs` / `epoch_end_abs_inclusive` / `epoch_complete`
- `ssvep3_pretrain_metadata_*.csv`
  - 移除 `target_samples` / `sample_adjustment`
  - 改为记录真实 `n_samples`

#### 1.4 数据集保存格式变更

```python
X = np.empty(len(self.dataset_x), dtype=object)
for i, epoch in enumerate(self.dataset_x):
    X[i] = epoch
```

数据集从固定长度 3D 数组改为一维 object array，支持变长 epoch。

### 3. CentralControllerSSVEPNode4.py - Decode 模式 EEG 采集闭环

#### 3.1 新增节点定位

`CentralControllerSSVEPNode4.py` 基于 `Node3` 复制扩展，主要用途是：
- 保留 `Node3` 的 pretrain 采集能力
- 为 decode 模式增加 EEG TCP 接收、trigger 打标、epoch 提取和 `.npy` 保存
- 统一 decode / pretrain 的通信配置加载逻辑

对应 ROS 入口：
```bash
ros2 run eeg_processing central_controller_ssvep_node4
```

#### 3.2 Decode 流程新增内容

在 decode 模式下：
- 仍然通过 `/image_seg` 发布 6 张图片和 `decode_prepare/decode_stim/decode_stop`
- 在进入 `decode_stimulating` 时发送 `trigger=1`
- 在停止刺激时发送 `trigger=2`
- 通过 EEG TCP 连续流内的 trigger 通道截取 decode epoch
- 保存：
  - `ssvep4_decode_mapping_*.csv`
  - `ssvep4_decode_trials_*.csv`
  - `ssvep4_decode_eeg_trials_*.csv`
  - `ssvep4_decode_metadata_*.csv`
  - `ssvep4_decode_dataset_*.npy`

decode 阶段新增一个短暂等待态 `decode_wait_capture`，用于在发送结束 trigger 后继续等待 EEG TCP 流中的 `trigger=2` 闭合当前 epoch。

#### 3.3 统一配置区重构

`Node4` 把原来散落在 decode / pretrain 两个分支中的通信参数抽到了统一配置区：
- Unity decode 回执 UDP：`decode_start_bind_ip` / `decode_start_port`
- Unity pretrain 回执 UDP：`train_trigger_bind_ip` / `train_trigger_bind_port`
- Windows trigger 转发：`trigger_local_*` / `trigger_remote_*`
- Windows EEG TCP：`eeg_server_*` / `eeg_recv_buffer_size` / `eeg_n_channels` / `eeg_frame_floats` / `eeg_fs`

同时新增内部配置加载函数：
- `_load_common_config()`
- `_load_shared_comm_config()`
- `_init_trigger_sender()`
- `_init_eeg_streaming()`
- `_init_decode_ack_receiver()`
- `_init_pretrain_ack_receiver()`

这样 decode 和 pretrain 共用一套通信配置，只保留时序参数差异。

### 4. validate_ssvep4_npy.py - Decode Epoch 绘图工具

新增 `validate_ssvep4_npy.py`，用途：
- 默认读取最新的 `ssvep4_decode_dataset_*.npy`

默认输出目录：
```text

```

## 架构演进说明

### Trigger 通道驱动 Epoch 的意义

| 方面 | 时间窗口截断 | TCP trigger 通道 |
|------|--------------|------------------|
| 时间精度 | 受 ROS 状态切换时延影响 | 由 EEG 采集流直接决定 |
| 数据完整性 | 可能截断边缘样本 | 保留完整 epoch |
| 变长支持 | 否 | 是 |
| decode / pretrain 一致性 | 否 | 是 |

### Node4 的新增意义

Node4 将 decode 从“只保存图片映射和 trial 元信息”升级为“同时保存对应 EEG epoch 数据”，使 decode 和 pretrain 都能产出统一格式的 `.npy` 数据集，便于后续可视化和离线分析。

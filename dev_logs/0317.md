## 核心文件变更详细说明

### 1. CentralControllerSSVEPNode3.py - Epoch 边界判定机制重构

#### 1.1 参数默认值调整

| 参数 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| `trigger_local_ip` | `"0.0.0.0"` | `"192.168.56.103"` | 绑定到特定网卡 IP |
| `eeg_fs` | `250.0` | `1000.0` | EEG 采样率调整为 1kHz |
| `eeg_frame_floats` 验证 | `>= eeg_n_channels` | `== eeg_n_channels + 1` | 必须包含 trigger 通道 |

#### 1.2 Epoch 边界判定机制变更（核心改动）

**旧方案**：基于时间窗口截断
- 在 `_enter_pretrain_stim()` 时记录 `stim_start_abs`
- 在 `_enter_pretrain_rest()` 时记录 `stim_end_abs`
- 使用 `_normalize_epoch_samples()` 对 epoch 进行截断/填充到固定长度 `target_samples`

**新方案**：基于 TCP 数据流中的 trigger 通道值
- 新增 `_process_eeg_trigger_sample()` 方法，从 TCP 数据流解析 trigger 值
- trigger 值语义：
  - `1`：epoch 开始边界
  - `2`：epoch 结束边界（inclusive）
- 移除 `target_samples` 字段和 `_normalize_epoch_samples()` 方法
- epoch 长度由实际 trigger 边界决定，不再强制对齐

#### 1.3 新增字段

```python
self.last_trigger_value = 0          # 上一个 trigger 值，用于边缘检测
self.epoch_start_pending = False     # 是否等待 epoch 结束标记
self.current_stim_enter_abs = -1     # 进入 stim 状态的 abs 索引
self.current_stim_exit_abs = -1      # 退出 stim 状态的 abs 索引
self.current_epoch_complete = False  # epoch 是否完成（收到 trigger=2）
self.current_trial_record_written = False  # trial 记录是否已写入
```

#### 1.4 CSV 字段更新

`ssvep3_pretrain_trials_*.csv` 和 `ssvep3_pretrain_meta_*.csv` 字段调整：

| 旧字段 | 新字段 | 说明 |
|--------|--------|------|
| `stim_start_abs` | `stim_enter_abs` | 进入 stim 状态时刻 |
| `stim_end_abs` | `stim_exit_abs` | 退出 stim 状态时刻 |
| - | `epoch_start_abs` | trigger=1 对应的 abs 索引 |
| - | `epoch_end_abs_inclusive` | trigger=2 对应的 abs 索引 |
| `target_samples` | - | 移除（不再强制固定长度） |
| `sample_adjustment` | `n_samples` | 实际样本数 |
| - | `epoch_complete` | epoch 是否完整 |

#### 1.5 数据集保存格式变更

```python
# 旧格式：固定长度 3D 数组
X = np.stack(self.dataset_x, axis=0)  # (n_epochs, n_channels, target_samples)

# 新格式：object array（支持变长 epoch）
X = np.empty(len(self.dataset_x), dtype=object)
for i, epoch in enumerate(self.dataset_x):
    X[i] = epoch
```

#### 1.6 Trial 记录写入时机调整

- 旧：在 `_enter_pretrain_rest()` 中写入 trial 记录
- 新：提取 `_write_pretrain_trial_row()` 方法，在以下时机调用：
  - `_start_pretrain_trial()` 开始下一个 trial 前
  - `destroy_node()` 节点销毁时

## 架构演进说明

### Epoch 边界判定的意义

| 方面 | 旧方案（时间窗口） | 新方案（trigger 通道） |
|------|-------------------|----------------------|
| 时间精度 | 受 ROS 回调延迟影响 | 由 EEG 采集端直接标记 |
| 数据完整性 | 可能截断有效数据 | 保留完整 epoch |
| 变长支持 | 否（固定长度） | 是（变长 epoch） |
| 同步可靠性 | 依赖 UDP trigger 外部同步 | TCP 数据流内置 trigger |

### 变长 epoch 的必要性

实际采集中，stimulus 呈现时间可能与预设值存在微小偏差：
- 本次采集数据：epoch 样本数范围 1519-1522（约 1.5-4ms 偏差 @ 1kHz）
- 强制截断会损失边缘样本或引入填充伪影
- 使用 object array 保存原始 epoch 更符合实际需求

3月17日的开发重构了 SSVEP3 预训练节点的核心 epoch 判定逻辑，从基于预设时间窗口改为基于 TCP trigger 通道值，显著提升了数据采集的时间精度和可靠性。同时修复了数据集验证脚本对 object array 格式的兼容性问题。
